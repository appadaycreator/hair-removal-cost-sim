# Hair Removal Cost Sim - 改善提案（P1-P4）

## 完成度評価（現状）
| カテゴリ | 完成度 | 理由 |
|---------|--------|------|
| **UI/UX** | 60% | デザインは良いが、部位ボタンのレイアウトが不安定。モバイルで行き当たりばったりの折り返し |
| **機能** | 65% | 基本計算は動作するが、保存・共有機能がない。FAQ が下部に埋もれている |
| **情報設計** | 70% | 必要な情報は揃っているが、ユーザーが検索しないと見えない |
| **コンバージョン** | 55% | アフィリエイト CTAは あるが、プリペイド系・クレジット決済などの選択肢がない |
| **パフォーマンス** | 85% | 高速（ブラウザ内計算）だが、Core Web Vitals 最適化が不明 |
| **アクセシビリティ** | 45% | カラーコントラスト不明、ARIA ラベルが最小限 |

**総合完成度**: **62%** → 実装後目標: **75-80%**

---

## 改善案（優先度順）

### 🔴 P1. 部位ボタンのグリッドレイアウト化とレスポンシブ化

**目的**: モバイル・PC での見栄えを改善し、ユーザー体験を統一する

**仕様**:
- 部位ボタンを固定 2 列グリッド（モバイル）→ 4 列グリッド（タブレット）→ 2-3 列（PC）に統一
- ボタンサイズを揃える（min-width: 100px、最小 48px 高さで TAP 対応）
- アクティブ状態（selected）をより目立たせる（背景色・アイコン・チェックマーク）

**実装タスク**:
1. `.part-btn` スタイルを grid ベースに修正
2. media query で列数を調整
3. `selected` クラスに チェックマーク（✓）アイコンか背景色のグラデーション追加
4. ボタンのタップターゲットを 48x48px 以上に確保

**DoD**:
- [ ] モバイル（375px）で 2 列グリッド、きれいに整列
- [ ] タブレット（768px）で 3-4 列、均等配置
- [ ] PC（1280px+）で見栄えが良い（横スペースを活用）
- [ ] selected ボタンがビジュアルに明確（✓マーク + 色付き）

---

### 🔴 P1. 期間・回数を大きく表示し、より目立つ「結果カード」に改善

**目的**: ユーザーが「何年かかる」「何回必要」を素早く理解できる

**仕様**:
- 結果セクションの上部に大型カード「脱毛に必要な期間・施術回数」を配置
- 現在の「合計費用」「月額」より手前に「期間」「回数」を配置
- 期間と回数をアイコン付きで分かりやすく表示

**実装タスク**:
1. 結果セクションの HTML 構造を修正（期間・回数を前に）
2. CSS で 大型・視認性の高いカード設計（フォントサイズ 20-24px、アイコン付き）
3. 期間・回数の値を calculate() で正しく供給

**DoD**:
- [ ] 期間（1-2年など）がデカく見える
- [ ] 施術回数が数字で目立つ
- [ ] モバイル・PC で見栄え一貫

---

### 🔴 P2. ページ読み込み時に FAQ を折りたたみ状態で「上部」に配置し、スクロール減少

**目的**: 「このツール本当に信用できるのか？」という不安を早期に解消

**仕様**:
- FAQ セクションを結果セクションより上に移動（もしくは sticky）
- 入力フォーム直下に 簡潔なミニ FAQ 3 問追加
- 「計算の仕組みは？」「データ送信される？」「医学的根拠は？」に答える

**実装タスク**:
1. HTML 構造で FAQ セクションを上に移動
2. ミニ FAQ を新規作成（details タグの <summary> で「よくある質問」）
3. 全体の情報フロー「入力 → ミニ FAQ → 計算 → 結果 → 詳細 FAQ」に整理

**DoD**:
- [ ] ミニ FAQ が入力フォーム直後に見える
- [ ] 詳細 FAQ は下部にあり、必要に応じてアクセス可能
- [ ] FAQ の内容が信頼感を高めている

---

### 🔴 P2. 全身・部位の共存ロジック改善＋ UI 警告

**目的**: 「全身」を選ぶと他の部位が自動クリアされる現仕様を、より明確に

**仕様**:
- 「全身」ボタンに警告アイコン（⚠️）を追加
- 「全身」クリック時、アラート or トースト「全身が選ばれました。他の部位は自動クリア」を表示
- 部位から「全身」に切り替え時のみアラート（最初の「全身」選択時は不要）

**実装タスク**:
1. togglePart() 関数に 「全身」から 「全身以外」への切り替え時のみアラート追加
2. トースト or alert() で「全身が選ばれました」メッセージを表示
3. ボタン UI に ⚠️ アイコン or 説明文を追加

**DoD**:
- [ ] 全身クリック時のビジュアルフィードバックがある
- [ ] アラート/トースト が うるさくない（重要な時だけ）
- [ ] ユーザーが「なぜ部位が消えた？」と混乱しない

### 🟡 P3. 医療脱毛 vs 光脱毛の「選択肢の根拠」を視覚化（表・比較表）

**目的**: ユーザーが選択肢の違いを科学的に理解できる

**仕様**:
- 脱毛タイプ選択直下に 比較表「医療脱毛 vs 光脱毛」を挿入
- 表: 永久性・痛み・回数・期間・効果・向き不向き をまとめる
- 表示/非表示はトグル（詳しく知りたい人向け）

**実装タスク**:
1. 比較表用の HTML テンプレート作成（<table> or カード並列）
2. 詳しく知りたい人向けの details タグで折りたたみ化
3. CSS で モバイル対応テーブル設計（横スクロール or カード形式）

**DoD**:
- [ ] 比較表がモバイル・PC で見やすい
- [ ] 表の情報が医療的に根拠のある内容
- [ ] ユーザーが「医療脱毛がいいのか光脱毛がいいのか」判断できる

## 資産価値評価

**現状**: ¥50〜80万円
- PV・動員力が中等（医療脱毛市場は需要あるが、競合多い）
- 専門知識が弱い（医学的根拠が不十分）
- Affiliate revenue が低い（リンク少ない）
- SEO: まあまあ（脱毛関連キーワード + affordance あるが、競合に負けやすい）

**完成後目標**: ¥100〜150万円（+50-60%）
- P1・P2 でユーザー体験向上 → PV + conversion rate ↑
- P3 で 医学的信頼感 ↑
- P4 で sticky user ↑（スクリーンショット保存で再訪問）
- Affiliate revenue 拡大（決済・ローン系で客単価 ↑）
